💰 Finance Tracker — System Architecture

Kotlin Multiplatform · Compose UI · Google Sheets Backend · Firebase Hosting

📱 Supported Platforms
Android (Primary) Web / WASM (Firebase Hosting) Desktop / JVM iOS (configured)
Single shared codebase via Kotlin Multiplatform — UI, ViewModels, data layer, and models are all shared in commonMain. Only HTTP engine and entry point differ per platform.

🧠 ViewModel & State
TransactionViewModel
Business logic, form validation, coroutine scope
TransactionUiState
StateFlow — loading, error, success, transaction list

MutableState — all form fields + dropdown visibility

🗄️ Data Layer
TransactionRepository
Orchestrates reads (Sheets API) and writes (Apps Script)
GoogleSheetsApi
GET transactions from Sheets REST API using API key
GoogleAppsScriptRepository
GET with query params to Apps Script URL — avoids CORS/POST redirect issues
Ktor HTTP Client
OkHttp (Android) · CIO (Desktop) · JS engine (Web)

Firebase Analytics + Auth

🔒 Security Measures
✅ API Key in local.properties
Gitignored — never pushed to repo
✅ Build-time injection
Keys baked into binary via BuildConfig, not in source
✅ Apps Script rate limit
100 req/hr per time window via CacheService
⚠️ Restrict API key
Recommended: limit to web.app referrer + Sheets API only
